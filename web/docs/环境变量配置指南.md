# 环境变量配置指南

本文档介绍如何通过环境变量配置数据库连接信息。

## 特殊字符处理

对于包含特殊字符（如 @、%、/ 等）的密码，系统会自动进行 URL 编码处理，确保特殊字符能够被正确解析。例如：
- 密码为 `my@password` 时，会被自动编码为 `my%40password`
- 密码为 `pass/word` 时，会被自动编码为 `pass%2Fword`

您无需手动对密码进行编码，直接设置原始密码即可。

## 基础环境变量 (Config)

以下环境变量适用于所有环境：

- `SNOW_APP_STATUS`: 运行环境，常用值为 `dev`、`stg`、`test`、`lite`
- `DB_DIALECT`: 数据库方言，默认为 "mysql"
- `DB_DRIVER`: 数据库驱动，默认为 "pymysql"
- `DB_USERNAME`: 数据库用户名，默认为 "root"
- `DB_PASSWORD`: 数据库密码，默认为 "root"
- `DB_HOST`: 数据库主机地址，默认为 "127.0.0.1"
- `DB_PORT`: 数据库端口，默认为 "3306"
- `DB_DATABASE`: 数据库名称，默认为 "snowball_data"

## 轻量环境变量 (LiteConfig)

轻量模式用于短期验证。默认使用 SQLite，并跳过 Redis、Dramatiq、APScheduler、flask-profiler。

### 数据库配置
- `SNOW_APP_STATUS`: 设置为 `lite`
- `LITE_DB_PATH`: SQLite 文件路径；不传时默认使用当前工作目录下的 `snowball_lite.db`

### 使用说明
- Lite 模式只保证最小启动链路
- 不依赖 Redis、异步任务队列和持久化 JobStore
- 如果要验证完整功能，请使用 `dev`、`stg` 或 `test`

## 开发环境变量 (DevConfig)

开发环境特定的环境变量：

### 数据库配置
- `DEV_DB_USERNAME`: 数据库用户名，默认为 "root"
- `DEV_DB_PASSWORD`: 数据库密码，默认为 "root"
- `DEV_DB_HOST`: 数据库主机地址，默认为 "localhost"
- `DEV_DB_PORT`: 数据库端口，默认为 "3306"
- `DEV_DB_DATABASE`: 数据库名称，默认为 "snowball"
- `DEV_DB_PROFILER`: 性能分析数据库名称，默认为 "snowball_profiler"

### Redis配置
- `DEV_REDIS_HOST`: Redis主机地址，默认为 "localhost"
- `DEV_REDIS_PORT`: Redis端口，默认为 "6379"
- `DEV_REDIS_DB`: Redis数据库编号，默认为 "1"
- `DEV_REDIS_PASSWORD`: Redis鉴权密码，默认为空（不使用鉴权）

## 使用说明

1. 在运行应用程序之前，请根据您的环境设置相应的环境变量。
2. 未设置的环境变量将使用默认值。
3. 建议将敏感信息（如密码）存储在环境变量中，而不是直接写在代码中。
4. 对于不同的部署环境，建议使用不同的环境变量配置文件。

## Redis鉴权配置示例

如果您的 Redis 服务器启用了密码鉴权，您可以通过设置相应的环境变量来提供密码：

### Windows
```cmd
set DEV_REDIS_PASSWORD=your_redis_password
```

### Linux/Mac
```bash
export DEV_REDIS_PASSWORD=your_redis_password
```

### Docker Compose
```yaml
services:
  web:
    environment:
      - DEV_REDIS_PASSWORD=your_redis_password
```

## 安全建议

1. 不要在代码仓库中存储包含实际密码的环境变量文件。
2. 在生产环境中使用强密码，并定期更新。
3. 限制数据库用户的权限，遵循最小权限原则。
4. 使用环境变量管理工具（如 dotenv）来管理环境变量。
5. Redis 服务在生产环境中应启用密码验证和适当的网络隔离措施。
