# 系统核心设计文档

本文档旨在记录 Snow View 项目中的关键架构决策和核心设计模式，为项目的维护和二次开发提供指导。

## 核心设计模式

### Pinia Store 的上下文无关使用 (`WithOut` 模式)

#### 1. 问题背景

Pinia 是一个基于 Vue 3 依赖注入系统的状态管理库。在标准用法中，调用 `useMyStore()` 必须在 Vue 组件的 `setup()` 上下文（或生命周期钩子）中进行，因为 Pinia 需要从中自动获取全局的 Pinia 实例。

然而，在项目的某些部分，我们需要在 Vue 组件上下文之外使用 Store，例如：

-   **路由守卫**: 在 `src/permission.ts` 中进行权限判断和动态路由生成。
-   **工具函数**: 在 `src/utils/` 目录下的某些纯 TS 模块中。
-   **Axios 拦截器**: 在请求或响应拦截器中需要访问 Store 状态。

在这些“无上下文”的环境中直接调用 `useMyStore()` 会导致 Pinia 无法找到活动的实例而报错。

#### 2. 解决方案：`WithOut` 封装

为了解决这个问题，项目采用了一种名为 `WithOut` 的封装模式。

**实现原理：**

1.  **创建并导出全局实例**: 在 `src/store/index.ts` 中，我们创建了一个全局唯一的 Pinia 实例并将其显式导出。
2.  **创建 `WithOut` 包装函数**: 在每个 Store 模块（例如 `user.ts`）的末尾，我们都提供了一个 `useUserStoreWithOut` 函数。
3.  **手动注入实例**: 这个 `WithOut` 函数在内部调用标准的 `useUserStore()` 函数，但它会**手动将第一步中导出的全局 Pinia 实例作为参数传入**。

```typescript
// src/store/modules/user.ts

import { defineStore } from 'pinia'
import { store } from '../index' // 导入全局实例

export const useUserStore = defineStore({
  id: 'user',
  // ...
})

// 包装函数，用于在组件外部调用
export const useUserStoreWithOut = () => {
  return useUserStore(store) // 手动传入 Pinia 实例
}
```

这个简单的包装绕过了 Pinia 的自动依赖注入，允许我们在任何地方安全地获取 Store 实例。

#### 3. 使用规范

-   **在 Vue 组件中**: **必须**使用标准的 `useMyStore()`。
    -   *示例*: 在 `.vue` 文件的 `<script setup>` 中。
-   **在 Vue 组件之外**: **必须**使用 `useMyStoreWithOut()`。
    -   *示例*: 在 `.ts` 模块中，如 `permission.ts`, `api/*.ts` 等。

遵循此规范可以确保代码的健壮性和可维护性。
