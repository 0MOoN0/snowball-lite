# Task 03：测试与验收口径更新（归档）

## 任务状态

- 状态：已完成
- 优先级：中
- 目标：把“根目录 web 必须存在”从当前测试和验收基线中移除
- 完成结果：monorepo 定向测试和当前有效基线说明都已改成以 `apps/backend/web` 为准

## 任务目标

这次任务只解决一件事：

- 更新测试和当前有效的基线说明，让它们不再把仓库根目录 `web` 当成必须存在的兼容事实

## 当前问题

当前至少有两类内容还把 `web` 兼容入口写成验收事实：

1. monorepo 阶段测试
2. monorepo / backend workspace 的阶段归档文档

这会导致一个问题：

- 即使代码和运行入口都已不依赖 `web`，测试和基线文档仍会把“删掉 `web`”判定成回归

## 任务范围

优先处理：

- `tests/test_monorepo_transition_task04.py`
- `tests/test_monorepo_transition_task05.py`
- 仍被当前 README 或阶段总结引用的基线文档

## 推荐方案

把验收口径从：

- “根目录 `web` 作为兼容入口继续存在”

调整为：

- “真实后端目录是 `apps/backend/web`”
- “运行和导入不再依赖根目录 `web`”
- “执行文档和长期文档继续按现有目录规则保留”

## 明确不做

- 不重写整个 monorepo 历史阶段文档
- 不把所有旧阶段测试一起重构
- 不删除根目录 `web`

## 验收标准

完成后至少满足：

- 关键 monorepo 测试不再断言根目录 `web` 必须存在
- 当前基线文档不再把根目录 `web` 写成现状事实
- 真实目录和兼容入口的边界在文档里表述一致

## 风险点

- 如果先删 `web` 再改测试，这一步会直接出现大面积假失败
- 如果测试改了但阶段文档没改，后续协作会继续混淆

## 推荐测试

- 定向跑 monorepo 相关测试
- grep 当前基线文档里的“根目录 `web` 是兼容入口”说法
