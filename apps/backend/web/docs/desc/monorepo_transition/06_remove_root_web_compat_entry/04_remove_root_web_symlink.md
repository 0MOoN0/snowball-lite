# Task 04：删除根目录 web 兼容入口并做最终清扫（归档）

## 任务状态

- 状态：已完成
- 优先级：高
- 目标：在前置任务完成后，正式移除仓库根目录 `web` 符号链接
- 完成结果：仓库根目录 `web` 已删除，lite/dev 导入 smoke 和 monorepo 定向测试已通过

## 前置条件

只有下面三件事都完成后，才能开始这个任务：

1. 运行与导入已经解耦
2. 活跃路径和工具配置已经迁移
3. 测试与验收口径已经更新

任何一项没完成，都不应该直接删 `web`

## 任务目标

这次任务只解决一件事：

- 正式删除仓库根目录 `web` 符号链接，并完成最后一轮清扫、验证和归档

## 任务范围

这一步应该做的事：

1. 删除根目录 `web` 符号链接
2. 再跑一轮 grep，确认活跃路径不再依赖 `web/...`
3. 跑定向测试和最小启动验证
4. 更新 requirement 状态和归档文档

## 最终验证重点

至少要确认：

- 日常启动命令仍可用
- pytest 关键路径仍可用
- 文档主入口不再引用根目录 `web` 作为当前事实
- 仓库根目录删除 `web` 后，没有新的导入错误

## 明确不做

- 不顺手改 Python 包名
- 不借机做第二轮 monorepo 大迁移
- 不把所有历史旧路径一次性清零

## 验收标准

完成后至少满足：

- 仓库根目录已经不存在 `web` 符号链接
- 当前活跃运行入口不再依赖根目录 `web`
- 当前活跃测试和文档不再依赖根目录 `web`
- 已补一份归档文档，明确说明兼容入口已移除

## 风险点

- 这是最后一步，最容易被低估
- 如果前面仍有漏网引用，这一步会把它们一次性放大成启动失败或测试失败

## 推荐测试

- 一条 lite 启动 smoke
- 一条 dev 启动或导入 smoke
- 一组 monorepo 定向测试
- 一轮针对 `web/...` 活跃引用的 grep
